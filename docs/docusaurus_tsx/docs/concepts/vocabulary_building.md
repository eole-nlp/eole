# Vocabulary Building

Vocabulary is a core component in neural machine translation and language models. EOLE provides multiple ways to build and manage vocabularies.

## Vocabulary Format

EOLE supports two vocabulary file formats:

### Format 1: Token with Count

```
<token>\t<count>
```

Example:
```
hello	100
world	50
test	10
```

- Each line contains a token and its frequency count, separated by a tab character (`\t`)
- The count is used for frequency-based filtering via `src_words_min_frequency` and `tgt_words_min_frequency`
- This format is generated by the `build_vocab` command

### Format 2: Token Only

```
<token>
```

Example:
```
hello
world
test
```

- Each line contains only a token
- No frequency information
- This format is commonly used when converting from HuggingFace models

### Format Detection

EOLE automatically detects the format by checking if the second field (after splitting by `\t`) is a valid integer:

```python
first_line = lines[0].split("\t", 1)
has_count = len(first_line) == 2 and first_line[-1].strip().isdigit()
```
